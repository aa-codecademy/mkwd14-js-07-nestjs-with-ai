Create a new account

New accounts are created with role: user by default. Promote via DB to admin.

Login

Returns accessToken (15m) + refreshToken (7d). Both are saved to localStorage so your session survives a page reload. Expired access tokens are refreshed automatically.

Forgot Password

Generates a 1-hour reset token. In production this would be emailed — here it's returned in the response so you can test it. Copy the resetToken from the output and use it in the Reset tab.

Reset Password

Paste the resetToken UUID from the Forgot Password response.

Refresh Token

Exchanges the stored refresh token for a brand-new access + refresh token pair (rotation — the old refresh token is invalidated immediately). User ID and refresh token are auto-filled from the current session.

Logout

Clears the stored refresh token hash in the database. Future refresh attempts with the old token will fail. The access token remains valid until it expires (15m) — this is an accepted trade-off for stateless JWTs.

Playlists

Any authenticated user can create playlists. You can only edit or delete your own playlists. Admins can edit/delete any playlist.

Create Playlist

You become the owner automatically. Only you (or an admin) can edit or delete it.

Replaces ALL songs in the playlist (PUT semantics). Only owner or admin.

Delete Playlist

Only owner or admin. Soft-deletes the record.
